Detailed Description

An implementation of a symmetric Cipher based Message Authentication Code generator

Example generating a MAC code from an Input array

CMAC mac(Enumeration::BlockCiphers::AHX);
SymmetricKey kp(Key);
mac.Initialize(kp);
mac.Update(Input, 0, Input.size());
mac.Finalize(Output, Offset);

Overview:

Cipher-based Message Authentication Code (CMAC), sometimes known as OMAC, is a block cipher-based message authentication code algorithm.
It can use any of the block ciphers in this library to provide assurance of message authenticity and the integrity of binary data.

Description:

Legend:
CIPH=encryption-function, K=key, b=block-size, M=message, K1,K2=subkeys, ^=XOR, ls=left-shift

Subkey Generation
1) Let L = CIPHK(0b).
2) If MSB1(L) = 0, then K1 = L ls 1;
Else K1 = (L ls 1) ^ Rb
3) If MSB1(K1) = 0, then K2 = K1 ls 1;
Else K2 = (K1 ls 1) ^ Rb.

MAC Function
1) Apply the subkey generation process to K to produce K1 and K2.
2) If Mlen = 0, let n = 1; else, let n = ⎡Mlen / b⎤.
3) Let M1, M2, ..., Mn - 1, Mn*, denote the unique sequence of bit strings such that M = M1 || M2 || ... || Mn - 1 || Mn*, where M1, M2, ..., Mn - 1 are complete blocks.
4) If Mn* is a complete block, let Mn = K1 ^ Mn*; else, let Mn = K2 ^ (Mn* || 10j), where j = nb - Mlen - 1.
5) Let C0 = 0b.
6) For i = 1 to n, let Ci = CIPHK(Ci - 1 ^ Mi).
7) Let T = MSBTlen(Cn).
8) Return T.

Implementation Notes:

  • Never reuse a ciphers key for the CMAC function, this is insecure and strongly discouraged.
  • MAC return size is the underlying ciphers block-size; e.g. for AES, 16 bytes, and can be truncated by the caller.
  • With the Initialize(Key) method, the key must be at least the ciphers block-size plus the minimum key size in length.
  • The Initialize(Key, Salt), and Initialize(Key, Salt, Info) methods, use the Key parameter as the cipher key, and the Salt as the initialization vector.
  • The Initialize(Key, Salt, Info) method assigns the Info array to an HX extended ciphers DistributionCode property; used by the secure key schedule to create an encryption domain.
  • After a finalizer call (Finalize or Compute), the Mac functions state is reset and must be re-initialized with a new key.

Guiding Publications:

  1. NIST SP800-38B: The CMAC Mode for Authentication.
  2. RFC 4493: The AES-CMAC Algorithm.
  3. NIST Rijndael ammended.
