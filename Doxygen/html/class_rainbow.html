Detailed Description

An implementation of the Rainbow asymmetric signature scheme (RAINBOW)

Key generation:

Rainbow sgn(RainbowParameters::RNBWS2S192SHAKE512);
IAsymmetricKeyPair* kp = sgn.Generate();
// serialize the public key
IAsymmetricKey* pubk = kp->PublicKey();
std::vector<byte> pk = pubk->ToBytes();

Sign:

Rainbow sgn(RainbowParameters::RNBWS2S192SHAKE512);
sgn.Initialize(PrivateKey);
std::vector<byte> msg(32);
std::vector<byte> sig(0);
// generate the signature
sgn.Sign(msg, sig);

Verify:

Rainbow sgn(RainbowParameters::RNBWS2S192SHAKE512);
sgn.Initialize(PublicKey);
std::vector<byte> message(0);
// authenticate the signature
if (!sgn.Verify(Signature, msg))
{
// authentication failed, do something..
}

Implementation Notes:

Rainbow is an asymmetric signature scheme based on multivariate polynomials over a finite field.
Signature sizes are small and vary by the the parameters; 127 bytes (S128), 156 bytes (S192), and 204 bytes with the strongest parameter S256.
Rainbow is designed to provide long-term security even against attackers equipped with quantum computers.

  • There are three available parameters set through the constructor, ordered by security strength (S1, S2, S3); RNBWS1S128SHAKE256, RNBWS2S192SHAKE512, and RNBWS3S256SHAKE512 (strongest)
  • The signature schemes operational mode (signing/verifying) is determined by the IAsymmetricKey key-type used to Initialize the cipher (AsymmetricKeyTypes: CipherPublicKey, or CipherPublicKey), Public for encryption, Private for Decryption.
  • The primary Prng is set through the constructor, as either an prng type-name (default BCR-AES256), which instantiates the function internally, or a pointer to a perisitant external instance of a Prng
  • Use the Generate function to create a public/private key-pair, and the Sign function to sign a message
  • The message-signature is tested using the Verify function, which checks the signature, populates the message array, and returns false on authentication failure
