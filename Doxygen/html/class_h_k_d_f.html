Detailed Description

An implementation of an Hash based Key Derivation Function (HKDF)

Generate an array of pseudo random bytes:

// use the enumeration constructor
HKDF kdf(Enumeration::Digests::SHA256);
// initialize
kdf.Initialize(Key, [Salt], [Info]);
// generate bytes
kdf.Generate(Output, [Offset], [Size]);

Overview:

HKDF uses an HMAC as a pseudo random function to produce pseudo-random output in a process known as key stretching.
HKDF has two primary functions; Expand, which expands an input key into a larger key, and Extract, which pre-compresses key and optional salt and info parameters into a pseudo random key.
The Extract step is called if the the KDF is initialized with the salt or info parameters, this compresses the input material to a key used by the HMAC.
The Info parameter may also be set via the Info() property, this can be used to bypass the extract step, while adding additional input to the HMAC compression cycle.
For best possible security, the Extract step should be skipped, the KDF is initialized with a key equal in size to the hash functions internal block-size, and the Info parameter is used as a secondary source of pseudo-random key input.
If used in this configuration, the Info parameter should be sized to the hash block-size, less one byte of counter, and any padding added by the hash functions finalizer.
Using this formula the HMAC is given the maximum amount of entropy on each expansion cycle, and the underlying hash function processes only full blocks of input.

Description:
Legend:
PRK=pseudorandom key, info=info-string, OKM=output-key, L=output-length

Extract:
PRK = HMAC-Hash(salt, Info)

Expand:
The output OKM is calculated as follows:
N = ceil(L/HashLen)
T = T(1) | T(2) | T(3) | ... | T(N)
OKM = first L octets of T
where:
T(0) = empty string (zero length)
T(1) = HMAC-Hash(PRK, T(0) | info | 0x01)
T(2) = HMAC-Hash(PRK, T(1) | info | 0x02)
T(3) = HMAC-Hash(PRK, T(2) | info | 0x03)
T(N) = HMAC-Hash(PRK, T(N) | info | N)
...
The constant concatenated to the end of each T(n) is a single octet counter.

Implementation Notes:

  • This implementation only supports the SHA2-256 and SHA2-512 message digests.
  • This class can be instantiated with a message digest or HMAC instance, or by using a digests enumeration type name.
  • The generator must be initialized with a key using one of the Initialize() functions before output can be generated.
  • The Initialize() function can use a SymmetricKey key container class, or input arrays of Key, and optional Salt and Info.
  • Initializing with a salt or info parameters will call the HKDF Extract function.
  • The Info parameter can be set via a property, bypassing the Extract step, and can be used as an additional source of entropy.
  • The recommended key and salt size is the digests block-size in bytes, the info size should be the block-size, less 1 byte of counter, and any padding added by the digests finalizer.
  • The minimum recommended key size is the digests output array size in bytes.

Guiding Publications:

  2. RFC 2104: HMAC: Keyed-Hashing for Message Authentication.
  3. RFC 5869: HMAC-based Extract-and-Expand Key Derivation Function.
