Detailed Description

An implementation of the Key Derivation Function: SCRYPT

Generate an array of pseudo random bytes:

// set to 10,000 rounds (default: 4000)
SCRYPT kdf(Enumeration::Digests::SHA256, 16384, 8, 1);
// initialize
kdf.Initialize(Key, [Salt]);
// generate bytes
kdf.Generate(Output, [Offset], [Size]);

Overview:

SCRYPT is a password-based key derivation function created by Colin Percival, originally for the Tarsnap online backup service.
SCRYPT uses a combination of a message digest and the Salsa stream cipher to make it costly to perform large-scale hardware attacks by requiring large amounts of memory to generate an output key.
Using the same input key, and optional salt, produces the exact same output.
It is recommended that a salt value is added along with the key.

Description:
Legend:
P=passphrase, S=salt, N=cpu/memory cost, r=block-size, P=parallelization parameter, DK=derived key

Generate:
1) Initialize an array B consisting of p blocks of 128 * r octets each:
B[0] || B[1] || ... || B[p - 1] = PBKDF2 - HMAC - SHA256(P, S, 1, p * 128 * r)
2) for i = 0 to p - 1 do: B[i] = scryptROMix(r, B[i], N)
3) DK = PBKDF2 - HMAC - SHA256(P, B[0] || B[1] || ... || B[p - 1], 1, dkLen)

Implementation Notes:

  • Class can be initialized with a message digest instance, or by using a digests enumeration type name.
  • The minimum key size is the size the digests return array in bytes, a key equal to the digests block size is recommended.
  • The use of a salt value can strongly mitigate some attack vectors targeting the key, and is highly recommended with SCRYPT.
  • The minimum salt size is 4 bytes, larger (pseudo-random) salt values are more secure.
  • The generator must be initialized with a key using one of the Initialize() functions before output can be generated.

Guiding Publications:

  1. SCRYPT: Stronger Key Derivation via Sequential Memory Hard Functions.
  2. RFC 7914: The scrypt Password-Based Key Derivation Function.
  3. Scrypt is Maximally Memory-Hard.

Parameters
DigestTypeThe hash functions type-name enumeral
CpuCostThe CPU cost parameter; increasing this value affects the cpu and memory cost.

This value must be evenly divisible by 1024, with the minimum legal size of 1024. The minimum recommended size is 16384.

Parameters
ParallelizationThe Parallelization parameter; indicates the number of threads used by the generator.

Change this value to multiply the cpu cost by this factor, the default value is 1. Setting this value to 0 will automatically use the number of system processor cores.

Exceptions
