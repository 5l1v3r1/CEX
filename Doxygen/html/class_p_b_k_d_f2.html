Detailed Description

An implementation of the Passphrase Based Key Derivation Version 2 (PBKDF2)

Generate an array of pseudo random bytes:

// set to 10,000 rounds (default: 4000)
PBKDF2 kdf(Enumeration::Digests::SHA256, 10000);
// initialize
kdf.Initialize(Key, [Salt], [Info]);
// generate bytes
kdf.Generate(Output, [Offset], [Size]);

Overview:

PBKDF2 uses an HMAC as a pseudo random function to process a passphrase repeatedly, producing pseudo-random output in a process known as key stretching.
By increasing the number of iterations in which the function is applied, the amount of time required to derive the key becomes more computationally expensive.
A salt value can be added to the passphrase, this strongly mitigates rainbow-table based attacks on the passphrase.

Description:
Legend:
DK=derived-key, c=iterations, hlen=digest-length, dkLen=output-length

Generate:
The function takes as parameters the passphrase, salt, the iterations count, and the output length. DK = PBKDF2(Password, Salt, c, dkLen).
DK = T1 || T2 || ... || Td klen/hlen
The function F is the XOR of (c) iterations of chained PRFs
The first iteration uses the password as the PRF key and salt concatenated with an incrementing counter (i).
Ti = F(Password, Salt, c, i)
Subsequent iterations use the passphrase as the key and the output of the previous computation as the salt.
U2 = PRF(Password, U1), U3 = PRF(Password, U2) ... Uc = PRF(Password, Uc-1).

Implementation Notes:

  • This class can be instantiated with a message digest or HMAC instance, or by using a digests enumeration type name.
  • The generator must be initialized with a key using one of the Initialize() functions before output can be generated.
  • The Initialize() function can use a SymmetricKey key container class, or input arrays of Key, and optional Salt and Info.
  • The minimum key (passphrase) size is 4 bytes, enforcing passwords of at least 8 characters is recommended.
  • The maximum number of bytes that can be generated is the digests return size * 255.
  • The use of a salt value can strongly mitigate some attack vectors targeting the passphrase, and is highly recommended with PBKDF2.
  • The minimum salt size is 4 bytes, larger (pseudo-random) salt values are more secure.
  • The default iterations count is 5000, larger values are recommended for secure server-side password hashing e.g. +100,000.

Guiding Publications:

  1. RFC 2898: Specification.
  2. RFC 6070: Test Vectors.
  3. NIST SP800-132: Recommendation for Password-Based Key Derivation.
