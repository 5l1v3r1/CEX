Detailed Description

The SHA2 configuration parameters structure

Contains the SHA2-256 and 512bit permutation functions.

The function names are in the format; Permute-rounds-bits-suffix, ex. PermuteR64P512C, 64 rounds, permutes 512 bits, using the compact form of the function.
The compact forms of the permutations have the suffix C, and are optimized for performance and low memory consumption (enabled in the hash function by adding the CEX_DIGEST_COMPACT to the CexConfig file).
The Unrolled forms are optimized for speed and timing neutrality (suffix U), and the vertically vectorized functions have the V suffix.
The H suffix denotes functions that take an SIMD wrapper class (ULongXXX) as the state values, and process state in SIMD parallel blocks.

This class contains wide forms of the functions; PermuteR64P8x512H and PermuteR80P4x1024H uses AVX2.
Experimental functions using AVX512 instructions are also implemented; PermuteR64P16x512H and PermuteR80P8x1024H.
These functions are not visible until run-time on some compiler platforms unless the compiler flag (AVX2 or AVX512) is explicitly declared.
