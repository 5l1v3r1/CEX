Detailed Description

An implementation of the Blake2B and Blake2BP sequential and parallel message-digests using a 512-bit hash code

Example using the simplified Compute method:

Use the Compute for small-to-medium sized data.

Blake512 dgt;
// compute a hash
dgt.Update(Input, 0, Input.size());
dgt.Finalize(Output, 0);

Implementation Notes:

  • The algorithm, parallel or sequential, is selected through the constructor (2B or 2BP); the parallel version is selected through either the Parallel flag, or via the BlakeParams ThreadCount() configuration parameter.
  • Parallel and sequential algorithms (Blake2B or Blake2BP) produce different digest outputs.
  • The sequential Block size is 128 bytes, (1024 bits), but smaller or larger blocks can be processed, for best performance, align message input to a multiple of the internal block size.
  • When using the Update function in a parallel configuration, input data length should be aligned to the ParallelBlockSize() for best performance, this is the minimum input size that triggers parallel processing.
  • Best performance for parallel mode is to use a large input block size to minimize parallel loop creation cost, block size should be in a range of 32KB to 24MB.
  • The number of threads used in parallel mode can be user defined through the BlakeParams->ThreadCount property to any even number of threads; note that hash output value will change with threadcount.
  • Digest output size is fixed at 64 bytes, (512 bits).
  • The ComputeHash(byte[], byte[]) function wraps the Update(byte[], size_t, size_t) and Finalize(byte[], size_t) functions; (suitable for small data)./>
  • The Update functions process message input, this can be a byte, 32–bit or 64-bit unsigned integer, or a vector of bytes.
  • The Finalize(byte[], size_t) function returns the hash code but does not reset the internal state, call Reset() to reinitialize to default state.
  • Setting Parallel to true in the constructor instantiates the multi-threaded variant.
  • Multi-threaded and sequential versions produce a different output hash for a message, this is expected.
  • Optional intrinsics are runtime enabled automatically based on cpu support.
  • SIMD implementation requires compilation with SSSE3 or higher.

Blake512::Blake512 ( BlakeParams &  Params)
explicit

Initialize the class with a BlakeParams structure.

The parameters structure allows for tuning of the internal configuration string, and changing the number of threads used by the parallel mechanism (ThreadCount). If the ThreadCount is greater than 1, parallel mode (Blake2BP) is instantiated. The default threadcount is 4, changing from the default will produce a different output hash code.

◆ ParallelProfile()

ParallelOptions & Blake512::ParallelProfile ( )
overridevirtual

Read/Write: Contains parallel settings and SIMD capability flags in a ParallelOptions structure.

The maximum number of threads allocated when using multi-threaded processing can be set with the ParallelMaxDegree(size_t) function. The ParallelBlockSize() property is auto-calculated, but can be changed; the value must be evenly divisible by the profiles ParallelMinimumSize() property. Note: The ParallelMaxDegree property can not be changed through this interface, use the ParallelMaxDegree(size_t) function to change the thread count and reinitialize the state, or initialize the digest using a BlakeParams with the FanOut property set to the desired number of threads.
