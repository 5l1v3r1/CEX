Detailed Description

An implementation of the Sphincs-Plus asymmetric signature scheme (SPHINCS+)

Key generation:

Sphincs sgn(SphincsParameters::SPXS1S128SHAKE);
IAsymmetricKeyPair* kp = sgn.Generate();
// serialize the public key
IAsymmetricKey* pubk = kp->PublicKey();
std::vector<byte> pk = pubk->ToBytes();

Sign:

Sphincs sgn(SphincsParameters::SPXS1S128SHAKE);
sgn.Initialize(PrivateKey);
std::vector<byte> msg(32);
std::vector<byte> sig(0);
// generate the signature
sgn.Sign(msg, sig);

Verify:

Sphincs sgn(SphincsParameters::SPXS1S128SHAKE);
sgn.Initialize(PublicKey);
std::vector<byte> message(0);
// authenticate the signature
if (!sgn.Verify(Signature, msg))
{
// authentication failed, do something..
}

Implementation Notes:

SPHINCS+ is a high-security post-quantum secure stateless hash-based signature scheme.
Signature sizes vary depending on the parameters; 16KB (S128), 33KB (S192), to 58KB with the strongest parameter S256.
Both the public and private keys are both less than 1KB in size.
SPHINCS+ is designed to provide long-term security even against attackers equipped with quantum computers.
Unlike most hash-based signature schemes, SPHINCS+ is stateless, allowing it to be a drop-in replacement for current signature schemes.

  • There are three available parameters set through the constructor, ordered by security strength (S1, S2, S3); SPXS1S128SHAKE, SPXS2S192SHAKE, and SPXS3S256SHAKE (strongest)
  • The ciphers operating mode (encryption/decryption) is determined by the IAsymmetricKey key-type used to Initialize the cipher (AsymmetricKeyTypes: CipherPublicKey, or CipherPublicKey), Public for encryption, Private for Decryption.
  • The primary Prng is set through the constructor, as either an prng type-name (default BCR-AES256), which instantiates the function internally, or a pointer to a perisitant external instance of a Prng
  • Use the Generate function to create a public/private key-pair, and the Sign function to sign a message
  • The message-signature is tested using the Verify function, which checks the signature and returns false on authentication failure
