Detailed Description

An implementation of the Skein sequential and parallel message-digests with a 256-bit hash code

Example using the Update method:

Skein256 dgt;
// compute a hash
dgt.Update(Input, 0, Input.size());
dgt.Finalize(Output, 0);

Tree Hashing Description:

The tree hashing mode is instantiated when the parallel mechanism is engaged through one of the two constructors parameters.
The default settings are applied when using the Skein256(bool) constructor, or a manually defined configuration when using the Skein256(SkeinParams) constructor.
The SkeinParams structure contains property accessors that are initialized by the boolean constructor at their defaults; Schema(83,72,65,51) OuputSize(64), Version(1), FanOut(Parallel ? 8 : 0), and LeafSize(32).
The SkeinParams structure when passed to the constructor, can be used to change the FanOut property (which corresponds to the number of threads used in parallel mode), which must be an even number less or equal to the number of processing cores.
For best performance in tree hashing mode, the message input block-size (Length parameter of an Update call), should be ParallelBlockSize in length.
The ideal parallel block-size is calculated automatically based on the hardware profile and algorithm requirments.
The parallel mode uses multi-threaded parallel processing, with each thread maintaining a single unique state.
The hash finalizer processes each leaf state as contiguous message input for the root hash; i.e. R = H(S0 || S1 || S2 || ...Sn).
The FanOut accessor in a SkeinParams structure is the sum number of leaves in the sequential hash chain, the default is 8, which uses eight states/threads.
Changing any of the SkeinParams values from their defaults, will produce a different hash output.
The SkeinParams structure also contains an optional DistributionCode() property, which is a personalization string applied when generating the initial state (the default is zero). The personalization string can be used to create a unique distribution, in either sequential or parallel operating modes. The DistributionCodeMax() accessor provides the upper limit to the personalization strings byte-length, this size is different for each variant; (256=8, 512=40, 1024=104).

Implementation Notes:

  • The input message block-size is 32 bytes, (256 bits).
  • Digest output size is 32 bytes, (256 bits).
  • The ComputeHash(byte[], byte[]) function wraps the Update(byte[], size_t, size_t) and Finalize(byte[], size_t) functions; (suitable for small data)./>
  • The Update functions process message input, this can be a byte, 32–bit or 64-bit unsigned integer, or a vector of bytes.
  • The Finalize(byte[], size_t) function returns the hash code but does not reset the internal state, call Reset() to reinitialize to default state.
  • Setting Parallel to true in the constructor instantiates the multi-threaded variant using a default FanOut of 8 threads.
  • Multi-threaded and sequential versions produce a different output hash for a message, and changing the Fanout property from the default of 8, will also change the output hash.
  • The supported tree hashing mode in this implementation is a sequential chain (hash list); intermediate hashes are finalized as contiguous message input to the root hash in the finalizer.

◆ ParallelProfile()

ParallelOptions & Skein256::ParallelProfile ( )
overridevirtual

Read/Write: Contains parallel settings and SIMD capability flags in a ParallelOptions structure.

The maximum number of threads allocated when using multi-threaded processing can be set with the ParallelMaxDegree(size_t) function. The ParallelBlockSize() property is auto-calculated, but can be changed; the value must be evenly divisible by the profiles ParallelMinimumSize() property. Note: The ParallelMaxDegree property can not be changed through this interface, use the ParallelMaxDegree(size_t) function to change the thread count and reinitialize the state, or initialize the digest using a SkeinParams with the FanOut property set to the desired number of threads.
