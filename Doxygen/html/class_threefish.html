Detailed Description

Contains the Threefish 256, 512, and 1024bit permutation functions.

The function names are in the format; Permute-rounds-bits-suffix, ex. PemuteR72P256C, 72 rounds, permutes 256 bits, using the compact form of the function.
The compact forms of the permutations have the suffix C, and are optimized for performance and low memory consumption (enabled in the hash function by adding the CEX_DIGEST_COMPACT to the CexConfig file).
The Unrolled forms are optimized for speed and timing neutrality (suffix U), and the vertically vectorized functions have the V suffix.
The H suffix denotes functions that take an SIMD wrapper class (ULongXXX) as the state values, and process state in SIMD parallel blocks.

This class contains wide forms of the functions; PemuteP4x256H, PemuteP4x512H, and PemuteP4x1024H use AVX2 instructions.
Experimental functions using AVX512 instructions are also implemented; PemuteP8x256H, PemuteP8x512H, and PemuteP8x1024H.
These functions are not visible until run-time on some compiler platforms unless the compiler flag (CEX_HAS_AVX2 or CEX_HAS_AVX512) is explicitly declared.

The compact form of the Threefish-1024 permutation function.

This function has been optimized for a small memory consumption. To enable this function, add the CEX_DIGEST_COMPACT directive to the CexConfig file. Note: The rounds count must be at least 72 and evenly divisible by 8.
