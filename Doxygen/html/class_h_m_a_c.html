Detailed Description

An implementation of a Hash based Message Authentication Code generator

Generating a MAC code

HMAC mac(Enumeration::BlockCiphers::AHX);
SymmetricKey kp(Key);
mac.Initialize(kp);
mac.Update(Input, 0, Input.size());
mac.Finalize(Output, Offset);

Overview:

A keyed Hash Message Authentication Code (HMAC) uses a cryptographic hash function with a secret key to verify data integrity and authenticate a message.
Any cryptographic hash function may be used in the calculation of an HMAC, including any of the hash functions implemented in this library. The cryptographic strength of the HMAC depends upon the strength of the underlying hash function, the size of its hash output, and on the size and quality of the key.

Description:

Legend:
H=hash-function, K=key, K'=derived-key, m=message, ^=XOR, ||=concatonate
Generate
Where opad is the outer padding (0x5c...5c), and ipad is the inner padding (0x36...36), and K' is a secret key, derived from key K.
HMAC(K,m) = H((K' ^ opad) || H(K' ^ ipad) || m))


Implementation Notes:

  • This implementation can utilize a parallelized digest instance for multi-threaded Mac calculations.
  • If the Parallel parameter of the constructor is set to true, or a parallelized digest instance is loaded, passing an input block of ParallelBlockSize bytes will be processed in parallel.
  • Sequential mode block size is the underlying hash functions internal block size in bytes.
  • Digest size is the hash functions output code size in bytes.
  • The key size should be equal or greater than the digests output size, and less or equal to the block-size.
  • The Compute(Input, Output) method wraps the Update(Input, Offset, Length) and Finalize(Output, Offset) methods and should only be used on small to medium sized data./>
  • The Update(Input, Offset, Length) processes any length of message data, and is used in conjunction with the Finalize(Output, Offset) method, which returns the final MAC code./>
  • After a finalizer call (Finalize or Compute), the Mac functions state is reset and must be re-initialized with a new key.

void HMAC::ParallelMaxDegree ( size_t  Degree)

Set the number of threads allocated when using multi-threaded tree hashing processing.

Thread count must be an even number, and not exceed the number of processor cores. Changing this value from the default (8 threads), will change the output hash value.

Parameters
DegreeThe desired number of threads
Exceptions
Exception::CryptoDigestExceptionThrown if an invalid degree setting is used
ParallelOptions & HMAC::ParallelProfile ( )

Read/Write: Contains parallel settings and SIMD capability flags in a ParallelOptions structure.

The maximum number of threads allocated when using multi-threaded processing can be set with the ParallelMaxDegree(size_t) function. The ParallelBlockSize() property is auto-calculated, but can be changed; the value must be evenly divisible by the profiles ParallelMinimumSize() property. Note: The ParallelMaxDegree property can not be changed through this interface, use the ParallelMaxDegree(size_t) function to change the thread count and reinitialize the state, or initialize the digest manually using a digest Params structure with the FanOut property set to the desired number of threads.
