Detailed Description

An implementation of the Key Derivation Function Version 2 (KDF2)

Generate an array of pseudo random bytes:

// set to 10,000 rounds (default: 4000)
KDF2 kdf(Enumeration::Digests::SHA256);
// initialize
kdf.Initialize(Key, [Salt], [Info]);
// generate bytes
kdf.Generate(Output, [Offset], [Size]);

Overview:

KDF2 uses a hash digest as a pseudo random function to produce pseudo-random output in a process known as key stretching.
Using the same input key, and optional salt and information strings, produces the exact same output.
It is recommended that a salt value is added along with the key, this mitigates some attacks against the function.

Description:
Legend:
Z=key, T=output-key, hlen=digest-length, kLen=output-length

Generate:
1) Set d = ceiling(kLen/hLen).
2) Set T = "".
3) for Counter = 1 to d-1 do:
C = IntegerToString(Counter, 4).
T = T || Hash(Z || C || [OtherInfo]).
4) Output the first kLen bytes of T as K.

Implementation Notes:

  • This implementation only supports the SHA2-256 and SHA2-512 message digests.
  • Class can be initialized with a message digest instance, or by using a digests enumeration type name.
  • The minimum key size is the size the digests return array in bytes, a key equal to the digests block size is recommended.
  • The use of a salt value can strongly mitigate some attack vectors targeting the key, and is highly recommended with KDF2.
  • The minimum salt size is 4 bytes, larger (pseudo-random) salt values are more secure.
  • The generator must be initialized with a key using one of the Initialize() functions before output can be generated.

Guiding Publications:

  1. ISO18033-2: Chapter 6.2.3 KDF2.
  2. RFC 6070: Test Vectors.
  3. NIST SP80056A: Recommendation for Pair-Wise Key Establishment Schemes: Chapter 5.8.
