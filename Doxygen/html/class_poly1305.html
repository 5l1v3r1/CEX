Detailed Description

An implementation of the Poly1305 Message Authentication Code generator

Example generating a MAC code from an Input array

// the default constructor uses the sequential processing mode (no cipher assist, just Poly1305)
Poly1305 mac(Enumeration::BlockCiphers::Rijndael);
// Note: if key is not pre-conditioned for Poly1305-AES, it will be clamped automatically in Initialize(&ISymmetricKey)
SymmetricKey kp(Key);
mac.Initialize(kp);
mac.Update(Input, 0, Input.size());
mac.Finalize(Output, Offset);

Overview:

Poly1305 and the cipher assisted variant Poly1305-AES are Message Authentication Code generators that return a 16-byte authentication code for a message of any length. Both variants use a a 32-byte secret key, with the Poly1305-AES mode also requiring a 16-byte nonce (unique message number).
This MAC generator can be used in tandem with a symmetric cipher, to generate an authentication code along with each encrypted message segment.

Implementation Notes:

  • This Mac has two modes of operation selected through the constructor; sequential, if no block cipher is selected, or cipher assisted (Poly1305-AES).
  • This Mac can use any one of the supported base block ciphers: Rijndael, Serpent, or Twofish. HX extended ciphers are not supported at this time.
  • With Poly1305-AES, the input Mac key is pre-conditioned to speed up multiplication by clearing required bits in the R portion of the key (first 16 bytes).
  • The Initialize(&ISymmetricKey) function tests the key for pre-conditioning, and if required, will clamp the key automatically..
  • Never reuse a nonce with the Poly1305 Mac, this is insecure and strongly discouraged.
  • MAC return size is 16 bytes, the array can be can be truncated by the caller.
  • The Initialize() function requires a key of 32 bytes (256 bits) in length.
  • After a finalizer call (Finalize or Compute), the Mac functions state is reset and must be re-initialized with a new key.

Initialize the MAC generator with a symmetric key container.

Uses a key, and optional info arrays to initialize the MAC. In a Poly1305-AES configuration, the R portion of the key is tested for pre-configuration and clamped automatically. The key size must be one of the block ciphers legal key sizes.
